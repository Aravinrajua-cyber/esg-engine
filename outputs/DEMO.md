# DEMO — 90-second rehearsable click path

**Goal:** one clean, memorized run. No improvising, no scrolling hunting for a name. Total ~90s.
**Pre-reqs:** site running (`scripts/start_site.bat` after `npm run build`), window at 1280px, demo
name locked (below), browser zoom 100%, cursor movements slow.

---

## Step 0 — Lock the demo name (do this once, after the Phase 4 freeze; ~30s of prep)
The Hidden Winner you search must be real and on-screen. After `companies.json` is frozen, pick the
top-ranked Hidden Winner with full data coverage:

```bash
cd /c/Hackathon/esg-engine && ./.venv/Scripts/python.exe -c "
import json
c = json.load(open('outputs/site_data/companies.json'))['companies']
hw = [x for x in c if x['classification']=='hidden_winner' and x['coverage_pct']>=80]
hw.sort(key=lambda x: x['rank'])
print('USE:', hw[0]['name'], '|', hw[0]['ticker'], '| rank', hw[0]['rank'], '| score', hw[0]['overall_score'])
"
```

Write the result here and rehearse with it: **`‹FROZEN: HIDDEN_WINNER_NAME / TICKER›`**.
Fallback if none clears 80% coverage: drop the threshold to 60%, or pick the highest-ranked
`hidden_winner` outright. Memorize the first 4–5 letters you'll type in search.

---

## The run (speak the **bold**; the rest is action)

**[0:00 — Leaderboard]** *Open already on the leaderboard.*
> "This is every ASEAN company we cover, ranked by ESG **momentum** — not level. Grade, a confidence
> band, and a data-coverage number on every single row."

*(Let it sit one beat. Don't scroll yet.)*

**[0:12 — Search]** *Type the first letters of the pre-picked name; click the row.*
> "Let me open a **Hidden Winner** — low ESG level, high ESG momentum. The names the static-score
> crowd hasn't repriced yet."

**[0:25 — Detail: explanation sentence]** *Detail view open; point at the explanation.*
> "This one sentence is generated by a **rule engine, not an LLM** — it names the two strongest
> positive drivers and the strongest negative one. You can trace the score to its inputs in one read."

**[0:40 — Risk flags + confidence band]** *Point at the flags, then the band.*
> "Risk flags with tooltips — controversy, liquidity, stale data. And the score isn't a point, it's a
> **band**: it widens when the evidence is thin. We show uncertainty instead of hiding it."

**[0:55 — Back to leaderboard]** *Click back.*
> "Back to the board — 477 names, all explainable the same way."

**[1:05 — Methodology tab → placebo chart]** *Navigate to Methodology; land on the placebo histogram.*
> "Before we trusted our own backtest, we tried to **kill** it: we shuffled the scores across
> companies a thousand times and re-ran everything. That's this hump at zero. Our real result is out
> here in the tail — p = **‹FROZEN: placebo_p›**."

**[1:20 — Close]** *Stop moving the cursor. Look up.*
> "This is **real data with real limits**. Our free ESG feed even died mid-build — and the model kept
> working on alternative data. **The limits are the story.**"

**[1:30 — END]**

## Rehearsal notes
- **If the site won't load:** fall back to the static figures — `money_chart.png` then
  `placebo_histogram.png` in `outputs/figures/` — and narrate the same beats. Practice this fallback.
- **Do not** open the weight sandbox in the 90s cut — it's a great answer to "isn't it just your
  weights?" in Q&A, but it eats time here. Keep it in your pocket.
- **Synthetic banner:** if running in synthetic mode the persistent banner is visible — name it
  ("this instance is on synthetic demo data; the live build runs the same code on real data") rather
  than letting a judge notice it first.
- **Two numbers to have cold:** the net Q5−Q1 spread and the placebo p-value. Everything else you can
  read off the screen.
- **Cadence:** 90 seconds is tight. The single most common failure is talking through the search box
  while typing — type silently, then speak on the click.
